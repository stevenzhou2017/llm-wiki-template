# Vibe Coding: 氛围编程

**来源：** [X/Twitter](https://x.com/karpathy/status/1886192184808149383)
**作者：** [[Andrej Karpathy]]
**日期：** 2025-02-02

## 定义

一种全新的编程方式：完全沉浸在"氛围"中，拥抱指数级效率，甚至忘记代码的存在。用自然语言（语音）告诉 LLM 要做什么，全程不看代码差异，直接接受所有更改。

## 核心特征

- **语音驱动**：通过 SuperWhisper 语音输入，几乎不碰键盘
- **工具**：Cursor Composer + Claude Sonnet
- **不读代码**：总是点击"全部接受"，不看 diff
- **错误处理**：遇到报错直接复制粘贴给 LLM，通常就能修复
- **绕过 bug**：LLM 修不了的 bug 就绕过去，或者随机改动直到消失

## 适用范围

作者明确说这适合"随手做的周末项目"（throwaway weekend projects），**不是**严肃工程的做法。

## 本质

> "I'm building a project or webapp but it's not really coding — I just see things, say things, run things, and copy paste things, and it mostly works."

这不是传统意义上的编程，而是一种新的人机交互模式：人负责意图和判断，LLM 负责实现。

## 影响

这条推文极度火爆，"vibe coding" 成为 2025 年 AI 编程领域的流行术语，引发了关于 [[AI辅助编程]] 未来方向的广泛讨论。
